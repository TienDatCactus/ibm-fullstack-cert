Functions là gì?
- Hệ quả của phương pháp phát triển phần mềm lập trình theo dạng module (modular programming SW methodology)
	- Tách chương trình thành nhiều modular components, mỗi phần chịu 1 trách nhiệm riêng
- Có cấu trúc, đơn lẻ và có thể dùng lại, thực hiện 1 chức năng / hành động cụ thể
- Một số ngôn ngữ refer tới chúng như là subroutines, procedures, methods hoặc module
- Cách hoạt động:
	- Nhận data làm input
	- Xử lí data
	- Trả về kết quả là output
- 2 loại functions:
	- Hàm thư viện tiêu chuẩn, built-in: `print, console.log,...`
	- Được định nghĩa bởi người dùng
	- Mỗi nnlt có kiểu xác định hàm khác nhau
- Cách sử dụng: create - invoke (gọi) - (khai báo)

Objects là gì?
- Cốt lõi trong OOP
- OOP là phương pháp lập trình tập trung vào các đối tượng (objects) thay vì functions
- Objects chứa dữ liệu ở dạng properties (attributes) (thuộc tính) và code dưới dạng procedures (methods) (phương thức)
- OOP đóng gói phương thức với cấu trúc dữ liệu -> Đối tượng sẽ làm việc với chính cấu trúc dữ liệu của nó 
-> Càng phức tạp về cấu trúc càng perform được nhiều chức năng
- Objects trong lập trình:
	- Chứa các trạng thái (properties) và hành vi (methods)
	- Lưu các properties trong các trường (variables)
	- Thể hiện hành vi thông qua methods (functions)
